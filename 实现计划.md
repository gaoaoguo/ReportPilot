# ReportPilot 实现计划

> 项目：AI 数据清洗与报表生成系统  
> 技术路线：Next.js + TypeScript + MySQL 8 + Prisma + Auth.js + Papa Parse + Recharts + DeepSeek  
> 开发环境：本地开发，不使用 Docker，本机 MySQL 8，本地文件存储  
> 文档目标：像盖房子一样，按“勘察 → 地基 → 主体 → 水电 → 装修 → 验收 → 扩建”的顺序渐进实现完整系统。

---

## 0. 总原则

### 0.1 先盖能住的房子，再装修豪宅

第一版不要追求功能多，而是追求完整闭环：

```txt
注册 / 登录
→ 创建 workspace
→ 上传 CSV
→ 本地保存文件
→ MySQL 记录文件与任务
→ worker 解析 CSV
→ 生成字段画像与数据质量检查
→ 调 DeepSeek 生成结构化报告
→ 保存报告
→ 前端展示报告
→ 用户只能看自己的数据
→ 错误可追踪
```

只要这个闭环跑通，你就已经从 demo 进入“完整系统”层级。

---

### 0.2 每一层都必须验收

不要一边写认证、一边写上传、一边写 AI、一边写图表。正确方式是：

```txt
一层完成
→ 自测
→ 写入 README / TODO
→ commit
→ 再进入下一层
```

每个阶段都有：

```txt
目标
前置条件
具体任务
验收标准
产物
风险点
建议 commit
```

---

## 1. 技术选择定稿

### 1.1 技术栈

| 模块 | 技术 | 用途 |
|---|---|---|
| Web 框架 | Next.js App Router | 页面、API、服务端逻辑 |

| 图表 | Recharts | 报告图表展示 |
| AI | DeepSeek API | 数据洞察与报告摘要 |
| AI SDK | openai npm package | 使用 OpenAI-compatible 格式接 DeepSeek |
| 校验 | Zod | API 入参、AI 输出校验 |
| Worker | tsx + MySQL jobs 表 | 本地异步任务处理 |
| 文件存储 | Node fs | 本地 `.local-storage/uploads` |
| 日志 | console + 自定义 logger | MVP 排错 |

### 1.2 DeepSeek 约定

使用环境变量控制模型：

```env
DEEPSEEK_BASE_URL="https://api.deepseek.com"
DEEPSEEK_API_KEY="your-api-key"
DEEPSEEK_MODEL_FAST="deepseek-v4-flash"
DEEPSEEK_MODEL_PRO="deepseek-v4-pro"
```

默认策略：

```txt
MVP 默认：deepseek-v4-flash
高质量重新生成报告：deepseek-v4-pro
```

AI 接入原则：

```txt
不要把整份 CSV 发给 AI
只发送字段画像、质量检查、少量脱敏样本
AI 输出必须是 JSON
AI 输出必须经过 Zod 校验
AI 失败不能导致系统崩溃
AI 调用必须记录日志
```

## 2.4 Auth.js 登录

### 2.4.1 配置 Auth.js

`src/lib/auth.ts`：

```ts
import NextAuth from "next-auth";
import Credentials from "next-auth/providers/credentials";
import { PrismaAdapter } from "@auth/prisma-adapter";
import { prisma } from "@/lib/prisma";
import bcrypt from "bcryptjs";

export const { handlers, auth, signIn, signOut } = NextAuth({
  adapter: PrismaAdapter(prisma),
  session: {
    strategy: "database"
  },
  providers: [
    Credentials({
      credentials: {
        email: {},
        password: {}
      },
      async authorize(credentials) {
        const email = String(credentials?.email || "").toLowerCase();
        const password = String(credentials?.password || "");

        const user = await prisma.user.findUnique({ where: { email } });
        if (!user?.passwordHash) return null;

        const valid = await bcrypt.compare(password, user.passwordHash);
        if (!valid) return null;

        return {
          id: user.id,
          email: user.email,
          name: user.name
        };
      }
    })
  ],
  pages: {
    signIn: "/login"
  }
});
```

API route：

```txt

```


## 2.5 权限工具

`src/lib/permissions.ts`：

```ts
import { auth } from "@/lib/auth";
import { prisma } from "@/lib/prisma";

export async function requireUser() {
  const session = await auth();

  if (!session?.user?.email) {
    throw new Error("AUTH_REQUIRED");
  }

  const user = await prisma.user.findUnique({
    where: { email: session.user.email }
  });

  if (!user) {
    throw new Error("AUTH_REQUIRED");
  }

  return user;
}

export async function getDefaultWorkspaceForUser(userId: string) {
  const membership = await prisma.workspaceMember.findFirst({
    where: { userId },
    include: { workspace: true },
    orderBy: { createdAt: "asc" }
  });

  if (!membership) {
    throw new Error("WORKSPACE_NOT_FOUND");
  }

  return membership.workspace;
}
```

## 4.4 Job 状态机

```txt
PENDING
  ↓
PROCESSING
  ├─ success → COMPLETED
  ├─ retryable error + attempts < maxAttempts → PENDING
  └─ fatal error / attempts exhausted → FAILED
```

## 4.5 Worker 基础版


必须：

```ts
where: {
  id: fileId,
  workspaceId: workspace.id
}
```

## 8.4 错误处理统一化

`src/lib/errors.ts`：

定义：

```ts
export class AppError extends Error {
  constructor(
    public code: string,
    public userMessage: string,
    public status = 400,
    public details?: unknown
  ) {
    super(userMessage);
  }
}
```

API 返回格式：

```json

  "error": {

    "message": "文件不能超过 10MB"
  }
}
```

用户不应该看到：

```txt

stack trace
SQL error
DeepSeek raw error
```

## 8.5 用量限制

添加模型：

```prisma
model UsagePeriod {
  id          String @id @default(cuid())
  workspaceId String

  periodStart DateTime
  periodEnd   DateTime

  filesProcessed   Int @default(0)
  rowsProcessed    Int @default(0)
  reportsGenerated Int @default(0)
  aiTokensUsed     Int @default(0)

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  workspace Workspace @relation(fields: [workspaceId], references: [id], onDelete: Cascade)

  @@unique([workspaceId, periodStart, periodEnd])
  @@index([workspaceId])
}
```

限制：

```txt
FREE_MAX_FILES=10
FREE_MAX_ROWS_PER_FILE=5000
FREE_MAX_AI_TOKENS_PER_MONTH=200000
MAX_UPLOAD_SIZE_MB=10
```

```txt

```
