- Workspace 模块：默认 workspace、成员关系、OWNER 权限。
- 页面模块：Landing、登录、注册、Dashboard、文件、上传、文件详情、报告、报告详情、设置。

- 文件仓库模块：CSV 校验、本地保存、文件元数据。
- 任务管线模块：ImportJob、worker 轮询、状态机、锁和重试。
- CSV 数据骨架模块：解析、字段类型识别、数据画像、质量检查、样本脱敏。
- DeepSeek AI 模块：client、prompt、JSON 输出、Zod 校验、AiCallLog、Report 入库。
- 报告和图表模块：报告列表、报告详情、质量评分、洞察卡片、Recharts 图表、字段画像表。

- 安全自检模块：权限隔离、路径安全、API key 暴露、AI 输入脱敏、错误栈隐藏。
- 文档交付模块：README、`.env.example`、测试 CSV 样本说明、启动和常见问题。

### 2. 页面清单

- `/`：Landing Page，引导注册。
- `/login`：登录页。
- `/register`：注册页。
- `/dashboard`：工作台，展示用量、最近文件、最近报告和上传入口。
- `/files`：文件列表页。
- `/files/new`：CSV 上传页。
- `/files/[fileId]`：文件详情页，展示状态、预览、字段画像和重新解析入口。
- `/reports`：报告列表页。
- `/reports/[reportId]`：报告详情页，展示摘要、质量评分、洞察、图表和建议。
- `/settings`：账号、workspace 和用量设置页。

### 3. API 接口清单

- `GET /api/health`：健康检查和数据库连通性检查。
- `POST /api/register`：用户注册并创建默认 workspace。
- `GET /api/files`：当前 workspace 文件列表。
- `POST /api/files`：CSV 上传、本地保存、创建 FileAsset 和 ImportJob。
- `GET /api/files/[fileId]`：文件详情。
- `GET /api/files/[fileId]/preview`：文件预览。
- `POST /api/files/[fileId]/parse`：重新创建解析任务。
- `GET /api/reports`：当前 workspace 报告列表。
- `GET /api/reports/[reportId]`：报告详情。
- `GET /api/usage`：当前 workspace 用量统计。
- `GET/POST /api/auth/[...nextauth]`：Auth.js 登录相关接口。

### 4. Prisma Model 清单

- `User`：用户。
- `Account`：Auth.js 账号。
- `Session`：Auth.js 会话。
- `VerificationToken`：Auth.js 校验 token。
- `Workspace`：工作区。
- `WorkspaceMember`：工作区成员与角色。
- `FileAsset`：上传文件元数据。
- `ImportJob`：导入和报告生成任务。
- `Report`：结构化报告。
- `AiCallLog`：DeepSeek 调用日志。
- `UsagePeriod`：用量周期。
- `AppEvent`：业务事件日志。

内部枚举使用英文稳定值：`WorkspaceRole`、`Plan`、`FileStatus`、`JobType`、`JobStatus`、`ReportStatus`、`AiProvider`、`AiModel`。

### 5. 角色权限清单

- 未登录用户：只能访问 Landing、登录、注册和公开静态资源。
- 登录用户：只能访问自己所属 workspace 的文件、任务、报告和用量。
- `OWNER`：MVP 中拥有当前 workspace 的全部权限。
- `ADMIN`：MVP 预留，暂不开放团队管理能力。
- `MEMBER`：MVP 预留，暂不开放团队协作能力。

所有 `file`、`job`、`report` 查询必须同时校验 `workspaceId` 和成员关系，禁止只按资源 ID 查询。

### 6. 核心业务流程闭环

```txt
用户注册
→ 密码哈希
→ 创建 User
→ 创建默认 Workspace
→ 创建 WorkspaceMember OWNER
→ 用户登录
→ 进入 Dashboard
→ 上传 CSV
→ 校验文件大小和类型
→ 保存到 .local-storage/uploads/{workspaceId}/{fileId}/original.csv
→ 写入 FileAsset
→ 创建 ImportJob
→ worker 领取 PENDING job
→ 标记 PROCESSING 并加锁
→ 解析 CSV
→ 生成 previewJson / rowCount / columnCount
→ 字段类型识别
→ 数据画像和质量检查
→ 样本脱敏
→ 调用 DeepSeek
→ JSON parse
→ Zod 校验
→ 记录 AiCallLog
→ 保存 Report
→ 更新 job 为 COMPLETED
→ 前端展示文件状态和报告详情
→ 用户可查询历史报告
```

### 7. 本阶段明确不做的功能

- 不做 Docker。
- 不做线上部署。
- 不做 Cloudflare R2 / S3。
- 不做 Stripe 支付。
- 不做 Redis / BullMQ。
- 不做 Excel 解析。
- 不做 PDF 导出。
- 不做 OAuth 登录。
- 不做团队邀请和复杂权限。
- 不做 Sentry、PostHog、邮件通知。
- 不做复杂后台管理系统。
- 不做完整 BI 编辑器。

### 8. 主要风险点

- 当前目录尚未初始化 Next.js 项目，后续 Phase 1 需要创建完整工程结构。
- 当前没有 `.env.example`、`README.md`、`prisma/schema.prisma` 和源码目录。
- 本地 MySQL 8 连接依赖用户本机环境，migration 需要用户准备数据库和 `.env.local`。
- DeepSeek 模型名和错误码可能随官方更新变化，环境变量必须可配置。
- CSV MIME 在不同浏览器中不稳定，必须同时校验扩展名、大小和内容。
- 文件路径必须由系统生成，不能使用用户原始文件名作为真实路径。
- worker 锁和状态机如果设计不严谨，可能重复处理任务。
- AI 输出可能不是合法 JSON，必须重试并用 Zod 校验。
- 推荐图表字段不能直接相信 AI，必须后端二次校验。
- 权限漏洞最可能出现在详情接口，只按 `fileId` / `reportId` 查询会串数据。
